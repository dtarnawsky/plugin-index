Web Native Plugins

This is a data source for Capacitor and Cordova plugins that have been tested to ensure they compile. Data is broken into single json files that are either lists of plugins or detailed information about a plugin.

Plugin Lists

Use the following links as a source of data:

  • plugins.json - A list of plugins with name, description and keywords
  • detailed-plugins.json - A list of plugins that includes all fields of the detailed plugin view (~1mb)
  • cap5-plugins.json - A list of plugins that build with Capacitor 5
  • cap4-plugins.json - A list of plugins that build with Capacitor 4
  • cap3-plugins.json - A list of plugins that build with Capacitor 3
  • cordova6-11-plugins.json - A list of plugins that build with Cordova (cordova-ios v6, cordova-android v11)

These are the fields from the plugin list

  • plugins - This an an array of plugin objects with the following fields:
  • name - This is the name as it can be used with npm. Ie install with npm install [name]
  • description - Short description of what the plugin does
  • keywords - This is a list of keywords taken from package.json in npm and topics from git info of url

These are the fields from the detailed plugin view

  • name - This is the name as it can be used with npm. Ie install with npm install [name]
  • version - This the last version number that was tested
  • success - This is the list of tests that succeeded.
  • repo - This is the git url to the source code (as described with npm view [name] --json)
  • author - The author (as found in package.json)
  • versions - This is a list of version numbers available
  • keywords - This is a list of keywords taken from package.json in npm and topics from git info of url
  • fails - This is the list of tests that failed.
  • bugs - Url of where to report bugs
  • description - Short description of what the plugin does
  • stars - Number of github stars if the plugin supplied a git repo
  • downloads - Number of monthly downloads on NPM
  • image - Url to the avatar of the publisher from npm
  • quality - A number which can be used to access how popular this plugin is (a combination of github issues, watchers and forks. If the repo is a fork it loses 100pts)
  • updated - Date Time the github repo was last updated (useful for identifying unmaintained plugins)
  • published - Date Time when the package was published to npm
